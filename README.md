# CLI Printing Press Library

The official collection of CLIs generated by the [CLI Printing Press](https://github.com/mvanhorn/cli-printing-press).

Every CLI in this library has been generated, verified (`printing-press verify`), and endorsed by the team behind the press. They're not wrappers - they're deep, opinionated tools with SQLite sync, offline search, workflow commands, and domain insights.

## What "Endorsed" Means

Every CLI in this library has passed:

1. **Generation** - Built by the [CLI Printing Press](https://github.com/mvanhorn/cli-printing-press) with all mandatory phases (0 through 5)
2. **Runtime Verification** - `printing-press verify` passes at 80%+ with data pipeline confirmed
3. **Scorecard** - Quality score of 75+ on the two-tier Steinberger scoring system
4. **Product Thesis** - A clear answer to "who is this for and why should they care"
5. **Emboss Pass** - At least one improvement cycle (`printing-press emboss`) applied

## Want Your CLI Listed Here?

If you've used the printing press to generate a CLI and want it in the library:

1. Generate your CLI with `/printing-press <API>`
2. Run `printing-press verify --dir ./your-cli --spec spec.json` - must pass at 80%+
3. Run `printing-press scorecard --dir ./your-cli` - must score 75+
4. Submit a PR adding your CLI as a subdirectory with:
   - The generated CLI source code
   - The spec used to generate it
   - The Phase 0-5 plan artifacts (in `docs/plans/`)
   - A verify report showing pass rate
   - A scorecard report showing quality score

### PR Review Criteria

We review every submission for:
- Does it actually work? (we run `printing-press verify` ourselves)
- Is the data layer real? (sync populates tables, search returns results, sql works)
- Are workflow commands useful? (not just API wrappers)
- Is the README a good pitch? (product thesis, not just command list)
- Does it pass with a real API key? (live testing, not just mock)

### What We Won't Accept

- CLIs that only wrap API endpoints without a data layer
- CLIs where `printing-press verify` fails
- CLIs generated from other tools (must be printing press output)
- CLIs for APIs that already have excellent official CLIs (check our [research](https://github.com/mvanhorn/cli-printing-press/blob/main/docs/plans/2026-03-27-research-top-50-api-candidates-plan.md))
